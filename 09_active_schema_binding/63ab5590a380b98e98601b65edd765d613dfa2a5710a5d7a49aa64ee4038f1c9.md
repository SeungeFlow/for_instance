# Canonical Track DB versus Mixed Directive Attempt Comparison

## 0. 비교목적

기존 Canonical Track DB와 혼합지시 방식의 최근 Attempt를 비교하여,
주제내용의 우열이 아니라 처리구조의 차이를 확인한다.

## 1. 비교군

```yaml
baseline_cells:
  - HRTDB_0002
  - HRTDB_0003
  role: CANONICAL_TRACK_DB_CELL_BASELINE

mixed_attempt:
  cycle: HRTDB_0004
  role: COMPARISON_ATTEMPT_A
  result_data_created: false
  state: HOLD_CORRECTION_REQUIRED
```

0004 Attempt는 삭제하지 않는다.
Function.2가 cross-seat dependency와 correction requirement를 탐지한 Formation Evidence다.

## 2. 현재 관측된 0004 특이점

```text
Expected Function.1.YZ direct pair
→ Data.Y + Data.Z

Observed dependency
→ Data.X secondary compatibility dependency

Result
→ Pair independence failure
→ Result.Data generation blocked
```

이 현상은 혼합된 문서·지시문 입력에서 경계혼입이 발생할 수 있다는 가설과 정합하지만,
단일 원인으로 확정하지 않는다.

## 3. 새 비교실험

새 주제는 기존 Track DB 주제와 무관한 주제로 선택한다.

```yaml
comparison_attempt_B:
  subject: NEW_UNRELATED_SUBJECT
  execution:
    one_subject_at_a_time: true
    one_stage_at_a_time: true
    one_directive_at_a_time: true
    index_before_directive: true
    anomaly_gate_after_every_output: true
```

기존 0004 Data를 재사용하는 실험과
완전히 새로운 주제로 새 Cell을 만드는 실험은 구분한다.

현재 우선순위:

```text
새 주제 1개
→ 적응형 11 Stage
→ 실제 Result.Data
→ Track DB Cell 승격판독
```
