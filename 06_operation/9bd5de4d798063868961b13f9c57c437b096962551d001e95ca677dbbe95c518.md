# Index-Ready-Directive-Anomaly Adaptive Protocol

## 0. 원칙

```text
한 번에 하나의 주제
한 번에 하나의 단계
한 번에 하나의 현재 상태
한 번에 하나의 개별지시문
```

## 1. 각 Stage의 표준 흐름

```text
1. 필요한 문서만 업로드
2. 인스턴스가 실제 File Byte를 읽음
3. 객체 Identity·시작·끝·직접입력·Context·제외범위 인덱싱
4. READY_FOR_DIRECTIVE 화면표시
5. gpt.xyzt가 인덱싱 상태 검수
6. 개별 인스턴스 지시문 1개 입력
7. 현재 Stage 출력 1개 생성
8. 특이점 화면표시
9. gpt.xyzt가 PASS/CORRECT/HOLD/STOP 판정
10. 다음 지시문 1개만 생성
```

## 2. Index Ready Surface

```yaml
index_ready:
  instance:
  seat:
  waiting_stage:
  indexed_objects:
  direct_input_objects:
  context_only_objects:
  excluded_objects:
  first_boundary:
  final_boundary:
  missing_objects:
  identity_verification_state:
  readiness: READY_FOR_DIRECTIVE
```

인덱싱은 실행이 아니다.

```text
INDEXED
≠ VERIFIED
≠ DIRECTIVE_ACTIVE
≠ PROCESSING_COMPLETE
```

## 4. 11개 적응형 Stage

| Stage | Seat | Active State | 선행 인덱싱 |
|---:|---|---|---|
| 01 | `gpt.x` | `DATA_X` | 새 주제·X Source 경계 |
| 02 | `gpt.y` | `DATA_Y` | 새 주제·Y Source 경계 |
| 03 | `gpt.z` | `DATA_Z` | 새 주제·Z Source 경계 |
| 04 | `gpt.xy` | `FUNCTION_1_XY` | `Data.X`, `Data.Y` |
| 05 | `gpt.xz` | `FUNCTION_1_XZ` | `Data.X`, `Data.Z` |
| 06 | `gpt.yz` | `FUNCTION_1_YZ` | `Data.Y`, `Data.Z` |
| 07 | `gpt.xyz` | `RESULT_XYZ` | Function.1 세 객체 |
| 08 | `gpt.xy` | `FUNCTION_2_XY` | Result와 자기 Function.1.XY |
| 09 | `gpt.xz` | `FUNCTION_2_XZ` | Result와 자기 Function.1.XZ |
| 10 | `gpt.yz` | `FUNCTION_2_YZ` | Result와 자기 Function.1.YZ |
| 11 | `gpt.xyz` | `RESULT_DATA_XYZ` | Result와 Review 세 객체 |

각 Stage 지시문은 직전 Stage의 실제 Output과 특이점 검수 후 생성한다.

## 7. Correction

Correction은 기존 Exact Object를 덮어쓰지 않는다.

```text
기존 Output
→ Formation / Singularity Track로 보존

Correction Directive
→ 새 Hash Object

Corrected Output
→ 새 Hash Object
```
