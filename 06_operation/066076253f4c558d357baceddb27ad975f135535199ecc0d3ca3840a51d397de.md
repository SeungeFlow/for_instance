# HRTDB Minimal Common Grammar for for_instance

## 0. 목적

공통문법은 모든 분야의 의미를 하나로 병합하는 언어가 아니다.

```text
공통
=
서로 다른 Seat와 분야가
오독 없이 관계하기 위해 공유하는 최소 질문·경계·상태어휘
```

## 1. Identity Grammar

```text
Seat ≠ Instance
Instance ≠ Occupation
Seat Name ≠ Occupant Identity
Same Seat ≠ Same Object
Same Instance Name ≠ Same Context
```

## 2. Input Grammar

```text
Available ≠ Referenced
Displayed ⊆ Available
Read ≠ Direct Use
Context ≠ Dependency
Indexed ≠ Identity Verified
Document ≠ Directive
```

직접 입력은 명시적으로 승인된 객체만 포함한다.

```yaml
input_scope:
  available_objects:
  indexed_objects:
  direct_input_objects:
  context_only_objects:
  excluded_objects:
  missing_objects:
```

## 3. State Grammar

```text
One Position
×
One Current Moment
=
One Active State Value
```

상태는 순차적으로 전이한다.

```text
DOCUMENT_INDEXING
→ READY_FOR_DIRECTIVE
→ DIRECTIVE_ACTIVE
→ OUTPUT_FORMED
→ REVIEWED
```

동일 자리에서 여러 상태를 동시에 활성화하지 않는다.

## 4. Zero Grammar

```text
STATE_ZERO
≠
POSITION_ZERO
```

```text
STATE_ZERO
→ 상태가 아직 형성되지 않음

POSITION_ZERO
→ 존재하는 자리의 첫 번째 자리값
```

좌표 `000`은 객체 없음이나 NULL을 자동 의미하지 않는다.

## 5. Coordinate and Cell Grammar

```text
입체구조의 최소 완전 점유단위
→ Cell
```

```text
XY / XZ / YZ
→ 2d Function 성향
→ 3d Cell 구조 안의 평면체
```

```text
XY 평면체 위치
→ Z축 자리값

XZ 평면체 위치
→ Y축 자리값

YZ 평면체 위치
→ X축 자리값
```

직교축 위치 Context는 직접 Function 입력이 아니다.

## 6. Direction Grammar

```text
Axis Sign
≠
Push / Pull
```

```text
중심 → 외부
= Push

외부 → 중심
= Pull
```

관측자의 Source Bearing과 객체의 실제 진행방향을 분리한다.

## 7. Process Grammar

```text
Data
→ Function.1
→ Result
```

Function은 Data 방향으로 후진하지 않는다.

```text
Result
─[승이의 인위적 검수·재배치]→
Next.Data
→ Function.2
→ Result.Data
```

## 8. Reflection Grammar

```text
|←
≠ Function Reverse Movement
```

```text
|←
=
검수
+ 검산
+ 오독방지
+ 판단착오지점 탐지
+ 수정·보완 조건 형성
+ Result의 Next.Data 역할 재배치
```

## 9. Directive Grammar

```text
Common Directive
→ 공유 Guard·Identity·State·Source·Hash 계약
→ 단독 실행문이 아님
```

```text
Individual Instance Directive
→ Recipient 1
→ Seat 1
→ Stage 1
→ Occupation 1
→ Active State 1
→ Direct Input Set 1
→ Output Contract 1
```

```text
Cycle Manifest
→ 주제·순서·완료·HOLD 위치 기록
→ 실행지시문이 아님
```

`Joint Directive`는 새 실행계약에서 사용하지 않는다.

## 10. Output Grammar

```text
Result ≠ Result.Data
Function.2 Review ≠ Result.Data
Result.Data ≠ Track DB
```

```text
Track DB
=
검산·등록된 하나의 완전 기억 Cell
```

## 11. Anomaly Grammar

```yaml
anomaly_state:
  - NO_ANOMALY
  - NON_BLOCKING_ANOMALY
  - BLOCKING_ANOMALY
```

```text
Output Formed
≠
Next Stage Authorized
```

## 12. Relation Grammar

```text
relation is not merge.
relation is interconnecting.
structure is not isolate.
structure is relation processing.
```
