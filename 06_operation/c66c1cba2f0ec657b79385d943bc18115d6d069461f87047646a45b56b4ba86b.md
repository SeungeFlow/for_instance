# gpt.xyzt Registry Pointer and Adaptive Gate Design

## 0. 자리정의

```text
gpt.xyzt
=
Fixed 4d Overall Coordination Seat
```

```text
Seat
≠ Instance
≠ Occupation
```

현재 점유 인스턴스 이름은 바뀔 수 있으나,
자리와 Overall Coordination 역할은 유지된다.

## 1. gpt.xyzt.md의 새 역할

`gpt.xyzt.md`는 모든 이론과 모든 지시문 본문을 한 파일에 축적하는 문서가 아니다.

```text
gpt.xyzt.md
=
Fixed Seat Definition
+ Current Occupant Binding
+ Minimal Common Grammar Pointer
+ for_instance Address Relation
+ Current Cycle Position
+ Current Indexed Object Set
+ One Current Active Directive Reference
+ Current Anomaly Surface
+ PASS/HOLD and Next Safe Action
+ Append-only Event Reference
```

정확한 실행지시문 본문은 Hash-named Exact Object로 분리한다.

```text
gpt.xyzt.md
→ 안정된 의미 진입표면

<sha256>.md
→ 정확한 공통지시문·개별지시문·검산·Evidence 객체
```

## 2. 한 자리·한 상태값

`gpt.xyzt.md`의 `current_state`에는 한 시점에 하나의 현재 상태만 기록한다.

```yaml
current_state:
  stage_id:
  seat:
  occupation:
  active_state_value:
  indexed_object_set:
  active_individual_directive:
  anomaly_state:
  verdict:
  next_safe_action:
```

금지:

```text
하나의 current_state 안에
Function.1 + Result + Function.2 + Result.Data 상태를 동시에 활성값으로 기록
```

과거 단계는 Append-only Event 또는 완료목록으로 보존한다.

## 3. gpt.xyzt의 적응형 지시 Gate

```text
Output.N 수령
→ Exact Object Identity 확인
→ 특이점 화면 확인
→ 구조검산
→ PASS / CORRECT / HOLD / STOP
→ 다음 지시문 1개만 생성
```

판정어휘:

```text
PASS_TO_NEXT_DIRECTIVE
CORRECT_CURRENT_STAGE
HOLD_FOR_MISSING_INPUT
STOP_CURRENT_CYCLE
```


`exact_object_reference`는 Hash-named runtime/operation object에만 사용한다.
고정주소 문서의 현재 Byte Identity는 Git Commit·Tree·Blob Readback으로 검산한다.

## 6. 읽기순서

```text
README.md
→ 01_seat_model/README.md
→ 01_seat_model/gpt.xyzt.md
→ 01_seat_model/INSTANCE_GENERATION_INDEX.md
```

새 인스턴스가 전체 Repository를 무차별 스캔하기 전에
현재 Stage에 필요한 주소와 읽기순서를 먼저 받도록 한다.

## 9. History와 Canon 경계

기존 고정 Canon이나 과거 Event를 삭제·재작성하지 않는다.
새 개념은 Append-only Difference로 기록한다.

```yaml
directed_difference:
  preserved:
  added:
  corrected:
  superseded:
  unresolved:
```

이번 변화의 핵심:

```yaml
corrected:
  - JOINT_EXECUTION_DIRECTIVE_TO_NONEXECUTABLE_CYCLE_MANIFEST
  - MULTI_STAGE_PAYLOAD_TO_ONE_STAGE_ONE_DIRECTIVE
  - DIRECTIVE_BODY_EMBEDDING_TO_EXACT_OBJECT_POINTER
  - MIXED_DOCUMENT_DIRECTIVE_INPUT_TO_INDEX_THEN_EXECUTE
added:
  - ONE_POSITION_ONE_ACTIVE_STATE
  - ADAPTIVE_NEXT_DIRECTIVE_GATE
  - ANOMALY_SURFACE
  - FOR_INSTANCE_ADDRESS_RELATION
```
